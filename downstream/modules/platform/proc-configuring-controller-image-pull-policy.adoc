= Configuring your controller image pull policy

[role="_abstract"]

Use this procedure to configure the image pull policy on your {ControllerName}.

.Procedure

. Log in to {OCP}.
. Go to menu:Operators[Installed Operators].
. Select your {OperatorPlatformNameShort} deployment.
. Select the *Ansible Automation Platform* tab.
. Click the {MoreActionsIcon} icon next to your {PlatformNameShort} instance and select btn:[Edit AnsibleAutomationPlatform].
. Click btn:[YAML view] and locate the `spec.controller:` section.
. Configure the image pull policy and resource requirements under the `controller:` section:
+
[source,yaml]
----
spec:
  controller:
    image_pull_policy: IfNotPresent  # Options: Always, Never, IfNotPresent
    image_pull_secrets:
      - pull-secret-name
    web_resource_requirements:
      limits:
        cpu: 1000m
        memory: 2Gi
      requests:
        cpu: 500m
        memory: 1Gi
    task_resource_requirements:
      limits:
        cpu: 2000m
        memory: 4Gi
      requests:
        cpu: 1000m
        memory: 2Gi
    ee_resource_requirements:
      limits:
        cpu: 500m
        memory: 1Gi
      requests:
        cpu: 250m
        memory: 512Mi
    redis_resource_requirements:
      limits:
        cpu: 500m
        memory: 1Gi
      requests:
        cpu: 250m
        memory: 512Mi
    postgres_resource_requirements:
      limits:
        cpu: 1000m
        memory: 2Gi
      requests:
        cpu: 500m
        memory: 1Gi
    postgres_storage_requirements:
      limits:
        storage: 10Gi
      requests:
        storage: 8Gi
    replicas: 1
    garbage_collect_secrets: false
    create_preload_data: true
----
+
. Click btn:[Save].
+
[NOTE]
====
These settings apply to the {ControllerName} component managed by this {PlatformNameShort} instance. If you specified an existing controller under `controller.name`, these settings will update that instance.

For more examples of {PlatformNameShort} custom resources, see link:{BaseURL}/red_hat_ansible_automation_platform/{PlatformVers}/html-single/installing_on_openshift_container_platform/index#appendix-operator-crs_operator-platform-doc[Appendix: {PlatformName} custom resources].
====
