= Prerequisites for content management

[role="_abstract"]
Before you can sync repositories and discover content in {SelfServiceShort}, you must configure authentication and ensure your repositories meet the required format and structure.

== Authentication requirements

To sync content from GitHub or GitLab repositories, you must:

* Create a Personal Access Token (PAT) with appropriate permissions for your source control provider
* Store the PAT securely in an OpenShift secret named `secrets-scm`
* Configure catalog locations in your Helm chart to point to your repositories

For GitHub, the PAT must have the following scopes:

* `repo`
* `read:org`
* `workflow` (optional, required only if syncing workflow templates)

For GitLab, the PAT must have the following scopes:

* `read_repository`
* `api`

== Repository format requirements

Repositories eligible for syncing must contain valid Backstage catalog entity files in YAML format. These files define:

* Execution environment definitions
* Ansible collections metadata
* Software templates
* Other discoverable content

Entity files must:

* Use `.yaml` or `.yml` file extensions
* Follow the Backstage catalog entity schema
* Include required metadata fields such as `apiVersion`, `kind`, `metadata.name`, and `spec`

== Security considerations

When managing content and authentication:

* Store PATs in OpenShift secrets, never in plain text or version control
* Use the minimum required token permissions for your use case
* Rotate PATs regularly according to your organization's security policy
* Review repository permissions to ensure only authorized users can modify synced content
* Consider using separate PATs for different repositories or environments
