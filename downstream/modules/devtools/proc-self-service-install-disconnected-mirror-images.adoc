= Mirror container images


. Log in to `registry.redhat.io`:
+
----
$ podman login registry.redhat.io
----
. Enter your Red Hat username and password when prompted.
. Log in to your disconnected registry:
+
----
$ podman login <disconnected_registry_url>
----
. Pull the {RHDH} image from `registry.redhat.io`:
+
----
$ podman pull registry.redhat.io/rhdh/rhdh-hub-rhel9:x.y.z
----
. Tag the image for your disconnected registry:
+
----
$ podman tag registry.redhat.io/rhdh/rhdh-hub-rhel9:x.y.z <disconnected_registry_url>/<your_namespace>/rhdh-hub-rhel9:x.y.z
----
. Push the tagged image to your disconnected registry:
+
----
$ podman push <disconnected_registry_url>/<your_namespace>/rhdh-hub-rhel9:x.y.z
----
. If you use OCI container delivery, mirror the Ansible plug-ins OCI artifacts image. Replace `<plugin-version>` with the version specified in `imageTagInfo` in your Helm chart configuration:
+
----
podman pull registry.redhat.io/ansible-automation-platform/automation-portal:<plugin-version>
podman tag registry.redhat.io/ansible-automation-platform/automation-portal:<plugin-version> <disconnected_registry_url>/<your_namespace>/automation-portal:<plugin-version>
podman push <disconnected_registry_url>/<your_namespace>/automation-portal:<plugin-version>
----
. If you use OpenShift image mirroring, configure the cluster to pull from your mirrored registry for `registry.redhat.io` content using an `ImageContentSourcePolicy` or equivalent configuration for your OpenShift version.
