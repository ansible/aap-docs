= Understanding the repository catalog

[role="_abstract"]
The repository catalog in {SelfServiceShort} provides a centralized view of all Git repositories configured for content synchronization.
The catalog displays repository status, sync history, and discoverable content such as execution environment definitions, collections, and software templates.

== Repository catalog functionality

The repository catalog provides a view of your configured Git repositories and enables you to manage content synchronization.

When a repository syncs successfully, {SelfServiceShort} discovers and catalogs entities defined in YAML files within the repository.
These entities become available in the appropriate catalog views.

== Synchronization options

Repositories can be synced in two ways:

* *Manual synchronization*: Trigger synchronization on demand
* *Automatic synchronization*: Configure repositories to sync on a schedule

Manual synchronization is useful when you add new content to a repository and want it available in the portal immediately.
