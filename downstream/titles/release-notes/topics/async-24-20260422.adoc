This release contains security updates and a set of bug fixes across multiple {PlatformNameShort} 2.4 components. 
This async release includes an {HubNameStart} enhancement for PQC-signed {ExecEnvShort}s, multiple fixes across {HubNameStart}, the {OperatorPlatformNameShort}, {ControllerName}, and {LightspeedShortName}, and a security fix for receptor.

== Enhancements

Added PCQ signature support so {HubNameStart} now supports {ExecEnvShort}s with PQC signatures.

== Bug fixes

=== {HubNameStart}


* Fixed an issue where container images in the {HubNameStart} UI could be incorrectly hidden when the `exclude_child_manifests` filter matched a parent manifest list in a different repository.(AAP-71824)


* Fixed an issue where downloading an invalid collection artifact could return an error due to `CollectionArtifactDownloadView`.`count_download` not providing name/namespace as expected.(AAP-71348)


=== {ControllerNameStart} and ee-supported 

* Fixed an issue in ee-supported where the `community.vmware` collection could be incompatible with Python 3.12 due to the `pyVmomi` dependency version requirement.(AAP-70105)

=== {OperatorPlatformNameShort}


* Fixed an issue where upgrading AAP 2.4 could fail during hub database migrations when `pulp_ansible` was upgraded from versions older than 0.23.0, by scaling down existing deployments and waiting for stale heartbeat records to expire before proceeding.(AAP-69743)


* Fixed an issue where `wisdom-manage` shell output could be incorrect after the Django 5.2 upgrade due to verbosity level changes.(AAP-69164)
