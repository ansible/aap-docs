== CVE

With this update, the following CVEs have been addressed:

* link:https://access.redhat.com/security/cve/cve-2025-66471[CVE-2025-66471] `automation-controller`: `urllib3` Streaming API improperly handles highly compressed data.(AAP-62007)

* link:https://access.redhat.com/security/cve/cve-2025-66471[CVE-2025-66471] `ansible-automation-platform-24/lightspeed-rhel8`: `urllib3` Streaming API improperly handles highly compressed data.(AAP-62004)

* link:https://access.redhat.com/security/cve/cve-2025-69223[CVE-2025-69223] `automation-controller`: AIOHTTP's HTTP Parser `auto_decompress` feature is vulnerable to zip bomb.(AAP-61853)

* link:https://access.redhat.com/security/cve/cve-2025-69223[CVE-2025-69223] `ansible-automation-platform-24/controller-rhel8`: AIOHTTP's HTTP Parser `auto_decompress` feature is vulnerable to zip bomb.(AAP-61840)

* link:https://access.redhat.com/security/cve/cve-2025-64460[CVE-2025-64460] `automation-controller`: `Django`: Algorithmic complexity in XML Deserializer leads to denial of service.(AAP-60953)

* link:https://access.redhat.com/security/cve/cve-2025-61729[CVE-2025-61729] `receptor`: Excessive resource consumption when printing error string for host certificate validation in `crypto/x509`.


== Deprecated

=== Execution Environment

`Openvswitch.openvswitch` collection has been deprecated for two years and is no longer certified. It has been removed from `console.redhat.com`, but is still available from link:https://galaxy.ansible.com/ui/repo/published/openvswitch/openvswitch/?version=2.1.1[Galaxy.ansible.com]. While supported it remains in `ee-supported` for {PlatformNameShort} 2.4 for a brief transition period, but will be removed in a future release.
