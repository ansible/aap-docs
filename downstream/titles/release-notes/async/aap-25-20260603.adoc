== Overview

This async release of Red Hat Ansible Automation Platform 2.5 delivers targeted bug fixes, security patches for multiple CVEs across platform components, and a platform installer rebase. Key areas of improvement include proxy compatibility for the automation calculator, performance gains for fact storage queries, resilience for transient HTTP errors in the automation controller collection, and critical vulnerability remediations in execution environments, automation hub, Ansible Lightspeed, the platform gateway UI, receptor, and controller dependencies. The minimum supported RHEL version is also updated to 9.6 for both the RPM and containerized installers.

== Bug fixes

=== {ControllerNameStart}

* Fixed an issue where analytics API requests did not respect proxy environment variables, causing DNS resolution failures in proxy environments. (AAP-75559)

* Fixed an issue where updating extra_data on a workflow job template node via PATCH failed with a labels validation error when the node had labels associated but the underlying Job Template had ask_labels_on_launch=False. (AAP-75203)

* Fixed an issue where the awxkit CLI failed on clean installs due to a missing packaging dependency in setup.py. (AAP-74279)

* Fixed an issue where long-running fact storage queries caused jobs and workflows to fail. (AAP-74521)

* Fixed an issue where the ansible.controller collection did not automatically retry transient HTTP errors (502, 503, 500, 504) with exponential backoff. (AAP-72704)


=== {ExecEnvNameStart}

* Fixed an issue where propcache >=0.5.0 was incompatible with setuptools < 82. (AAP-75088)


=== {GatewayStart} UI

* Fixed an issue where searching for a collection by name returned incorrect results. (AAP-65974)


=== Containerized installer {PlatformNameShort}

* Fixed an issue where uploading large images to automation hub returned a "502 Bad Gateway" error. (AAP-55134)
