== Overview

This {PlatformNameShort} 2.5.20260422 async release includes a small set of product improvements (including Automation Hub PQC signature verification), a deprecation notice related to operator migration secrets, multiple security fixes (CVEs) across several AAP components, and bug fixes focused on stability, performance, and UI responsiveness.


== Enhancements


=== {HubNameStart}


* Added PCQ signature support so Automation Hub now supports Execution Environments with PQC signatures.(AAP-71607)

== Bug Fixes


=== {PlatformNameShort} UI

* Fixed an issue where the Jobs list page generated excessive API requests due to unthrottled WebSocket refresh events, which could result in queued requests and an unresponsive UI on systems with many concurrent jobs (AAP-70350).

* Fixed an issue where the Jobs list page generated excessive API requests due to unthrottled WebSocket refresh events, which could result in queued requests and an unresponsive UI on systems with many concurrent jobs (AAP-66302).


=== Content

* Fixed an issue where the ansible.controller collection job_template module did not support Bitbucket (AAP-71826).


=== {ControllerNameStart}


* Fixed an issue where variables from set_stats artifacts were passed via extra_vars and could take higher precedence than artifacts set by later nodes in the workflow chain (AAP-70755).

* Fixed an issue where creating an object could take excessive time in an organization with many resources due to RoleEvaluation object creation overhead (AAP-70753).

* Fixed an issue where an inventory sync with a large amount of changes could take an excessive amount of time (AAP-70376).

* Fixed an issue in ee-supported where the pyVmomi dependency in the community.vmware collection was not compatible with Python 3.12 (AAP-70105).

* Fixed an issue where concurrent jobs could incorrectly clear host facts due to a race condition (AAP-69263).

* Fixed an issue where canceling did not propagate to dependent jobs in workflows (AAP-68974).

* Fixed an issue where project_update.yml could fail with a Jinja2 error when using custom execution environment images with newer ansible-core versions (AAP-68784).


=== {EDAName}

* Fixed an issue where a project could be deleted while a project sync was running (AAP-71407).

=== {HubNameStart}

* Fixed an issue where CollectionArtifactDownloadView.count_download did not provide name and namespace instead of filename (AAP-71347).


=== {OperatorPlatformNameShort}

* Fixed an issue where wisdom-manage shell command output was incorrect due to a verbosity-level change (AAP-69164).
