== Overview

This release of {PlatformNameShort} 2.5 delivers a new analytics logging feature, performance and metadata enhancements for the automation controller, an EDA role-assignment API, updated {ExecEnvShort} collections, and security fixes addressing multiple CVEs across controller, hub, receptor, {ExecEnvShort}, and gateway components. Bug fixes improve stability for EDA activations, documentation rendering, RBAC permission grants, and inventory host counting.

== Highlights

* New controller logging of ingress API response metadata for analytics upload traceability
* RBAC query performance optimization for non-superuser API requests at scale
* EDA API endpoint for role team assignment
* Security fixes for 17 CVEs across receptor, hub, controller, {ExecEnvShort}, lightspeed, and gateway
* Stability improvements for EDA rulebook activations on Kubernetes

== Features

=== {ControllerNameStart}

* The controller now logs the `request_id`, `account_number`, and `org_id` from the ingress API response after a successful `gather_analytics` tarball upload. Previously, this response data was silently discarded, requiring support engineers to manually patch the controller source code to trace analytics uploads through Kibana. The new log entry appears at INFO level in `task_system.log` and can be searched directly by `request_id` or `account_number` to correlate controller uploads with backend ingestion. (AAP-79938)

== Enhancements

=== {ControllerNameStart}

* Installed collections and Ansible version are now recorded for every job run, regardless of whether the Indirect Node Counting feature flag is enabled. Previously, this metadata was only collected when FEATURE_INDIRECT_NODE_COUNTING_ENABLED was on. (AAP-74343), (AAP-79224)
* Performance: Optimized RBAC permission filtering for non-superuser API requests to /api/v2/unified_jobs/ and /api/v2/unified_job_templates/. Replaced an inefficient 3-table JOIN in UnifiedJobTemplate.accessible_pk_qs() with a direct subquery using _actor_role_filter(), eliminating an unnecessary scan of the dab_rbac_objectrole table. This reduces query latency for non-superusers, particularly at scale. (AAP-70902) (AAP-78349)

=== {EDAName}

* API Endpoint for Role Team Assignment for EDA. (AAP-80332)

== Bug fixes

=== {GatewayStart}

* Fixed an issue where a potential Subject header spoofing vulnerability existed on gateway routes. (AAP-79294)

=== {GatewayStart} UI

* Fixed an issue where the collection Documentation tab in {HubName} would crash when the API response included large unnecessary fields, causing the page to fail to render. (AAP-75538)

=== {ControllerNameStart}

* Fixed an issue where organization project admins could not assign sibling teams permissions on projects. (AAP-48417)
* Fixed an issue where a non-admin user with "can add notification template" permission received an Internal Server Error when creating a notification template, due to an AttributeError in give_creator_permissions preventing the DAB RBAC assignment from succeeding without a corresponding old-style role. (AAP-77110)
* Fixed an issue where after controller-task pod recreation, the control node was not re-added to the controlplane, causing all jobs to remain indefinitely pending until manual instance-group adjustments were performed. (AAP-78338)
* Fixed an issue where constructed inventories would increase the host count in the main dashboard. (AAP-80063)

=== {EDAName}

* Fixed an issue where automatic sync was not triggered after editing a project in EDA. (AAP-63051)
* Fixed an issue where PostgreSQL logs were flooded with "could not receive data from client: Connection reset by peer" messages. Adds setting for db connection to maintain/reuse the connection for up to 10 minutes to reduce noise in the logs. (AAP-77001)
* Fixed an issue where sandboxed Jinja2 rendering was not enforced in EDA Server. (AAP-77443)
* Fixed an issue where transient Kubernetes API errors could cause rulebook activations to enter an unrecoverable error state, leading to orphaned activation pods and duplicate pod spawning. Activations now retry on transient errors and automatically recover when the API becomes available again. (AAP-81709)

=== {ExecEnvNameStart}

* Fixed an issue where KubeVirt inventory sync was failing. Updated the following collections: kubernetes.core 5.4.0, redhat.openshift 4.0.2, redhat.openshift_virtualization 1.5.0. (AAP-49501)
* Fixed an issue where collections were outdated. Updated the following collections: ansible.platform 2.7.20260615, ansible.controller 4.8.2, ansible.eda 2.12.1. (AAP-79065)

=== Hub

* Fixed an issue where deleting a community.general collection caused excessive memory usage, leading to SIGKILL. (AAP-53296)
