Use a default runtime environment
Ansible Automation Platform includes the following default runtime environments, or execution environments, which you can specify when setting up a job template in the Ansible Automation Platform UI or access them from automation hub.

While supported execution environments cover many automation prerequisites, minimal execution-environments are the recommended basis for your own custom images, to keep full control over dependencies and their versions.
ee-supported - Contents of the minimal execution environment, plus all Red Hat-supported collections and dependencies.
You can set a global default execution environment for Ansible Automation Platform, or you can set one per job template.
Set a global default execution environment
To use supported execution environments, you can configure a specific execution environment globally, or you can set one per job template. 

To globally set a default execution environment for all jobs:

From the Ansible Automation Platform UI, go to Settings > Automation Execution > System. 
Click Edit.
From the Global default execution environment drop-down, select the default execution environment you want to set.
Click Save.
Set a default for a job template
To use supported execution environments, you can configure a specific execution environment globally, or you can set one per job template. 

To set an execution environment for specific automation jobs:
From the Ansible Automation Platform UI, go to Automation Execution > Templates.
Click the job template you want to configure. For example, click Demo Job Template.
Click Edit template and specify the Execution environment you want to use. By default, this is set to Default execution environment.
Click the execution environment name to see its details, including the location of the execution environment container image. For the default, the image resides at registry.redhat.io.
